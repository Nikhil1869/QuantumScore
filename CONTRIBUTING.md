# Contributing to QuantumScore

Thanks for your interest in contributing! Here's how to get started.

## Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/<your-username>/QuantumScore.git
   cd QuantumScore
   ```
3. **Create a branch** for your feature or fix:
   ```bash
   git checkout -b feature/my-feature
   ```

## Development Setup

Follow the [Quick Start](README.md#quick-start) section in the README to set up all three services (ML, Backend, Frontend).

## Making Changes

- Keep commits focused — one logical change per commit.
- Write clear commit messages (`fix: …`, `feat: …`, `docs: …`).
- Add or update tests where applicable.
- Make sure all services still start correctly after your changes.

## Submitting a Pull Request

1. Push your branch to your fork:
   ```bash
   git push origin feature/my-feature
   ```
2. Open a **Pull Request** against the `master` branch of this repository.
3. Describe what your PR does and link any related issues.
4. Wait for review — we'll get back to you as soon as possible.

## Code Style

| Language | Style |
|----------|-------|
| Python | PEP 8 |
| JavaScript | ESLint defaults / Prettier |
| CSS | Consistent with existing `globals.css` |

## Reporting Issues

Open a [GitHub Issue](https://github.com/Nikhil1869/QuantumScore/issues) with:
- A clear title and description
- Steps to reproduce (if it's a bug)
- Expected vs. actual behavior
